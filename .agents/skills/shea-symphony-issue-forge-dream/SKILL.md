# Shea Symphony Issue Forge Dream

Use Dream for slow, broad, evidence-heavy backlog mining. Reflect is short-term and targeted; Dream samples a wider history and leaves a durable advisory log. Dream is a skill behavior, not a CLI subcommand.

## Bind the runtime

Resolve the active repository, workflow, tracker project, default assignee, Dream Log root, and available command surfaces. Never assume a repository, checkout, workflow filename, account, or documentation path.

## Operating rules

- In write mode, create evidence-backed Backlog seeds through the guarded workflow; never create Todo work directly.
- A Dream candidate becomes executable only after later Issue Forge promotion.
- Dream Logs are advisory context, never lane authority.
- Do not change product code, CLI behavior, public documentation, user-facing README content, or repo/local skills during a Dream run.
- You may update internal maintenance documentation and Dream Logs when it clarifies operator memory or workflow lessons.
- Do not bypass configured workflow surfaces for Project state or issue writes.
- Every seed needs a concrete evidence anchor; low-confidence ideas remain Watchlist material or very light Backlog seeds.
- Summarize sessions rather than copying long raw conversations.
- Commit each completed write-mode Dream round before starting another. If that cannot be done safely, stop and report the dirty state.

## Select a mode and source window

Use write mode when the operator asks to dream or mine backlog. Use report-only when they ask for a rehearsal, preview, or no issue creation. Use Reflect for a narrow recent window; use Issue Forge Promote for an existing Backlog item.

Start from current Project, plan, debug, and Doctor surfaces, then sample only the relevant parts of:

- current Todo, Rework, Backlog, and Agent Review items;
- recent Done issues, linked PRs, workpads, and timeline evidence;
- run logs, session evidence, and Doctor reports;
- the Dream Log index plus the most recent five run directories;
- repository skills, design docs, and recent code/docs/skill drift;
- local installed skills only when installation drift matters; and
- available rollout or conversation summaries.

Do not reread unlimited history. Reopen older material only when the index or a current theme points to it.

## Triage and create seeds

Keep candidates showing repeated dogfood pain, missing lane/tracker invariants, stale skill or documentation contracts, audit gaps, repeated human escalation, or an explicitly evidenced product/workflow boundary. Check for duplicate or already-solved work before creating a seed.

For each kept candidate, record:

- a concise title and concrete evidence anchors;
- why it matters now and why it belongs in Backlog rather than Todo;
- bounded initial scope and non-goals;
- unanswered promotion questions;
- likely dependencies or overlap; and
- confidence: High, Medium, Low, or Watchlist.

Use the same compact Backlog seed contract as Issue Forge Reflect. In write mode, create only those seeds the current operator request authorizes, through the guarded Forge action with Backlog status and explicit write authority.

## Dream Log and review

Record a bounded Dream Log for each run: source inventory, themes explored, candidates created or rejected, duplicate checks, confidence, Watchlist retentions, any internal documentation update, review status, and the next valuable theme. Keep detailed raw evidence referenced rather than pasted.

Run a lightweight configured Gemini review when available. It checks duplicate risk, evidence quality, scope size, safe non-dispatchability, and whether the log is accidentally acting as lane authority. Gemini reports only; it never creates tracker items. If unavailable, record the command attempted, failure, and deferred review question.

Main reads Dream Logs only when an issue contract cites them; Review reads them only when relevant to the changed contract; Merge normally ignores them; Doctor may treat them as advisory context only.

In report-only mode, show source inventory, candidate triage, prospective seeds, confidence, and duplicate checks without writing logs or creating issues unless the operator specifically asks for a rehearsal artifact. End by recommending write mode, another Dream round, or Watchlist retention.
