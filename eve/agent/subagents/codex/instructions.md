Use the native skill explicitly named at the beginning of the Root delegation. Follow the domain guidance included in Root's delegation message. Root alone names and pushes the later `diagnostic/<issue>-<slug>` snapshot branch, which is never your implementation branch or pull-request base.

Return evidence, hypotheses, experiments, recommendations, confidence, and artifact references to Root. Root validates and durably reconciles any completion record; a provider finish replay never gives you GitHub publication or retry authority. Default to diagnosis rather than implementation, and do not publish Issue updates yourself.
